> **Difficulty:** Medium  
> **Topics:** `Greedy` `Array`

## Problem Statement

You are given a **0-indexed** array of integers `nums` of length `n`. You are initially positioned at index `0`.

Each element `nums[i]` represents the **maximum jump length** from index `i`.

Return the **minimum number of jumps** required to reach index `n - 1`.

> **Note:** It is guaranteed that you can always reach the last index.

## Constraints

```text
1 <= nums.length <= 10⁴
0 <= nums[i] <= 1000
It's guaranteed that you can reach the last index.
```

---

# Approach: Greedy (Range Expansion)

Instead of deciding **which index to jump to**, think of every jump as covering a **range**.

For every index inside the current reachable range:

- Calculate the farthest position that can be reached.
- Continue exploring until the current range ends.
- Once the range is exhausted, take one jump.
- The next range becomes the farthest position found.

This guarantees the minimum number of jumps.

---

## Algorithm

1. Initialize

```text
jumps = 0
currentEnd = 0
farthest = 0
```

2. Traverse from `0` to `n - 2`.

3. Update

```text
farthest = max(farthest, i + nums[i])
```

4. If

```text
i == currentEnd
```

- Increment `jumps`
- Set

```text
currentEnd = farthest
```

5. Return `jumps`.

# Python Solution

```python
class Solution:
    def jump(self, nums):
        jumps = 0
        current_end = 0
        farthest = 0

        for i in range(len(nums) - 1):
            farthest = max(farthest, i + nums[i])

            if i == current_end:
                jumps += 1
                current_end = farthest

        return jumps
```

---

# Java Solution

```java
class Solution {
    public int jump(int[] nums) {

        int jumps = 0;
        int currentEnd = 0;
        int farthest = 0;

        for (int i = 0; i < nums.length - 1; i++) {

            farthest = Math.max(farthest, i + nums[i]);

            if (i == currentEnd) {
                jumps++;
                currentEnd = farthest;
            }
        }

        return jumps;
    }
}
```

# Why Greedy Works

Suppose the current jump allows us to reach every index inside a range.

Instead of immediately jumping from the first index, we inspect every index in that range and compute how far each one can take us.

The index that gives the **maximum future reach** defines the next range.

Because every jump is postponed until the current range is fully explored, we never waste a jump.

This guarantees the minimum number of jumps.

---

# Key Insight

> We **don't choose the next index**.  
> We **choose the farthest reachable range** for the next jump.

This transforms a problem that looks like Dynamic Programming into an elegant **O(n) Greedy** solution.
