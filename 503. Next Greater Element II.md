# 503. Next Greater Element II

## Problem

Given a **circular integer array** `nums`, return the **next greater element** for every element.

- The array is **circular**, meaning after the last element, traversal continues from the beginning.
- The next greater element of `x` is the **first greater element encountered while moving forward**.
- If no greater element exists, return `-1`.

---

## Example 1

**Input**

```text
nums = [1,2,1]
```

**Output**

```text
[2,-1,2]
```

**Explanation**

- First `1` → Next greater is `2`
- `2` → No greater element → `-1`
- Second `1` → Search continues from beginning → `2`

---

## Example 2

**Input**

```text
nums = [1,2,3,4,3]
```

**Output**

```text
[2,3,4,-1,4]
```

---

# Algorithm (Monotonic Stack)

Since the array is **circular**, traverse it **twice**.

1. Let `n = len(nums)`.
2. Initialize answer array with `-1`.
3. Create an empty stack to store **indices**.
4. Traverse from `0` to `2*n - 1`.
5. Current index = `i % n`.
6. While the stack is not empty and the current element is greater than the element at the top index:
   - Update answer for the top index.
   - Pop the stack.
7. Push indices only during the **first traversal** (`i < n`).
8. Return the answer array.

---

# Python Code

```python
class Solution(object):
    def nextGreaterElements(self, nums):
        """
        :type nums: List[int]
        :rtype: List[int]
        """
        n = len(nums)
        ans = [-1] * n
        stack = []

        for i in range(2 * n):
            while stack and nums[stack[-1]] < nums[i % n]:
                ans[stack.pop()] = nums[i % n]

            if i < n:
                stack.append(i)

        return ans
```

# Complexity Analysis

| Operation | Complexity |
|-----------|------------|
|Traversal|O(2n) = O(n)|
|Stack Operations|O(n)|
|Overall Time|**O(n)**|
|Space|**O(n)**|

---

# Key Idea

- Use a **Monotonic Decreasing Stack** that stores **indices**.
- Traverse the array **twice** to simulate circular traversal.
- Each index is pushed only once and popped at most once.
- This guarantees an efficient **O(n)** solution.
