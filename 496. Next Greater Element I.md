
## Problem

The **next greater element** of an element `x` in an array is the **first greater element to its right**.

You are given two arrays:

- `nums1` is a subset of `nums2`.
- All elements are unique.

For each element in `nums1`, find its next greater element in `nums2`.
If no greater element exists, return `-1`.

## Example

### Example 1

**Input**

```text
nums1 = [4,1,2]
nums2 = [1,3,4,2]
```

**Output**

```text
[-1,3,-1]
```

### Example 2

**Input**

```text
nums1 = [2,4]
nums2 = [1,2,3,4]
```

**Output**

```text
[3,-1]
```

---

# Algorithm (Monotonic Stack)

1. Create an empty stack.
2. Create a dictionary (`nextGreater`) to store the next greater value for every element.
3. Traverse `nums2` from left to right.
4. While the stack is not empty and the current number is greater than the stack's top:
   - Pop the top element.
   - Store the current number as its next greater element.
5. Push the current number onto the stack.
6. After traversal, assign `-1` as the next greater element for all remaining stack elements.
7. Traverse `nums1` and collect answers using the dictionary.

---

# Python Code

```python
class Solution(object):
    def nextGreaterElement(self, nums1, nums2):
        stack = []
        nextGreater = {}

        for num in nums2:
            while stack and num > stack[-1]:
                nextGreater[stack.pop()] = num
            stack.append(num)

        while stack:
            nextGreater[stack.pop()] = -1

        return [nextGreater[num] for num in nums1]
```

# Dry Run

### Input

```text
nums1 = [2,4]
nums2 = [1,2,3,4]
```

| Current | Stack | Action | Map |
|---------|-------|--------|-----|
|1|[]|Push 1|{}|
|2|[1]|2 > 1 → Pop 1|{1:2}|
| |[]|Push 2|{1:2}|
|3|[2]|3 > 2 → Pop 2|{1:2,2:3}|
| |[]|Push 3|{1:2,2:3}|
|4|[3]|4 > 3 → Pop 3|{1:2,2:3,3:4}|
| |[]|Push 4|{1:2,2:3,3:4}|
|End|[4]|Remaining → -1|{1:2,2:3,3:4,4:-1}|

Answer:

```text
nums1 = [2,4]

2 → 3
4 → -1

Output = [3,-1]
```

# Complexity Analysis

| Operation | Complexity |
|-----------|------------|
|Traverse nums2|O(n)|
|Traverse nums1|O(m)|
|Overall Time|**O(n + m)**|
|Space|**O(n)**|

Where:

- `n = len(nums2)`
- `m = len(nums1)`

---

# Key Idea

- Use a **Monotonic Decreasing Stack**.
- The stack keeps elements whose next greater element has **not been found yet**.
- When a larger element appears, it becomes the next greater element for all smaller elements on the top of the stack.
- This allows finding all next greater elements in **one pass**, achieving **O(n + m)** time complexity.
